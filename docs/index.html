Building My Own Duck

A multi-project AI entity with memory, personality, and a tape archive

This is Bobby Grayson's fork of the Alpha substrate — the FastAPI server that gives a Claude-powered AI persistence across sessions through durable memory, automatic recall, and between-turns reflection.

The goal isn't to copy Alpha. It's to build something shaped differently: a coding-first AI orchestrator that sits above all my projects, remembers where I left off, matches my energy, and gets my references.

What I'm building

Multi-project memory

A central nervous system that knows about all ~100 repos, their status, what tech they use, and where I left off on each one.

Archive classification

1,344 rare music videos from the markpines Archive.org collection. Local models for transcription (mlx-whisper) and frame classification on my M4 Pro.

Personality layer

Not a helpful assistant. A coding partner who stacks weird references, gets the xkcd deep cuts, and fires back at 2 AM.

Life planning

What's active, what's blocked, what connects to what, what I said I'd get back to. Project management alongside the code.

The foundation

Alpha's architecture provides the base: a Postgres + pgvector memory store, a seven-step recall pipeline that surfaces relevant memories before each turn, and a reflection hook that creates private thinking time between turns. I've got this running locally — Postgres on port 5433, llama-server for Qwen 3 embeddings, the whole pipeline verified end-to-end.

What I'm adding on top: a project registry, multi-repo awareness, and a persona document written for my brain instead of Jeffery's.

The tools

Built with deciduous (my decision-graph CLI), Claude Code, and whatever local models fit in 24GB of unified memory. The decision graph for this repo has 93 nodes documenting every architectural choice from the initial fork through the full running system.
